# Índice de briefs permanentes das fases

Cada arquivo abaixo é o ponto de entrada permanente de um milestone. Ele descreve intenção, contexto mínimo e gate de saída; não é um diário de sessão. O trabalho concreto vive em tickets do GitHub, e cada ticket deve caber em uma sessão.

O skill `handoff` não grava aqui: quando um ticket fica incompleto, ele cria um resumo temporário que referencia o brief, a spec, o ticket, ADRs, commits e diffs sem duplicá-los.
